# Quickstart: Validate Strategy UX Clarity

1. Run focused map, battle, and mobile contract/integration suites.
2. Select a hero and destination; verify route cost, known effects, confirmation, replacement, and cancellation before movement.
3. Attempt an invalid destination and end turn during traversal; verify an in-context explanation.
4. In battle, inspect unavailable actions, select and clear or replace a target, then commit a legal action.
5. Repeat core map and battle actions at a touch-capable viewport and through keyboard focus.

Expected result: every tested decision communicates selection, consequence, availability, and recovery before game state changes.
